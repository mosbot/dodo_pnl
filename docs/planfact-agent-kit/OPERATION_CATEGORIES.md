# ПланФакт — Учётные статьи (Plan of Accounts) для AI-агента

Этот файл фиксирует иерархию учётных статей, их тип и семантические правила для агента.

`README.md` — policy, router.
`LOOKUPS.md` — как получить `operationCategoryId` через API.
`USE_CASES.md` — что входит в «выручку», «прибыль» и другие составные метрики.
`OPERATIONS.md` — как работать с `operationParts` и `operationCategoryType`.

## Когда открывать

- нужно понять, к какому типу (Income / Outcome / Assets / Liabilities / Capital) относится статья;
- нужно определить scope «выручки», «расходов», P&L или баланса;
- нужно знать, денежная или неденежная статья, и какие даты доступны;
- нужно понять, является ли статья системной (🔒) или скрытой из UI (🚫).

## Quick-reference: тип статьи и участие в отчётах

Если нужен только тип статьи и её участие в отчётах — достаточно этой таблицы. Полная иерархия с системными флагами — ниже.

| `operationCategoryType` | Рус. название  | В ОПУ?               | Статья участвует в ДДС? | В Балансе?    | Даты в операции                  |
| ----------------------- | -------------- | -------------------- | ----------------------- | ------------- | -------------------------------- |
| `Income`                | Доходы         | ✅ доходная часть     | ❌                       | ❌             | Дата оплаты + Дата начисления    |
| `Outcome`               | Расходы        | ✅ расходная часть    | ❌                       | ❌             | Дата оплаты + Дата начисления    |
| `Assets`                | Активы         | ❌                    | ✅ (некоторые)           | ✅             | зависит от подтипа статьи        |
| `Liabilities`           | Обязательства  | ❌                    | ✅ (некоторые)           | ✅             | Денежная — только дата оплаты    |
| `Capital`               | Капитал        | ❌ (кроме Дивидендов) | ✅ (кроме Дивидендов)   | ✅             | зависит от подстатьи             |

> ⚠️ Колонка «Статья участвует в ДДС?» — о типе **статьи**, не о типе **операции**. Операции типа `Income` (Поступление) и `Outcome` (Выплата) всегда попадают в ДДС независимо от `operationCategoryType`. Тип статьи влияет на то, в какую группу ДДС строка попадает, но не на сам факт включения. Не путать: `operationType` (Income/Outcome/Move/Accrual/Shipment/Supply) ≠ `operationCategoryType` (Income/Outcome/Assets/Liabilities/Capital).

**Выручка по умолчанию** = `1.1 Нераспределённый доход` + `1.2 Выручка от продаж`. Не включать `1.3 Прочие доходы` без явного запроса.

**Прибыль** = только статьи `Income` − статьи `Outcome`. Assets / Liabilities / Capital (кроме Дивидендов) — не участвуют.

**ДДС** строится по **типу операции**, не по типу статьи: Поступления + Выплаты + Перемещения. Начисления, Отгрузки, Поставки — не участвуют.

## Общая классификация счетов

### Активные счета

Все статьи **Актива**, **Расхода** и **Капитал/Дивиденды**.

### Пассивные счета

Все статьи **Обязательств**, **Капитала** (кроме Дивидендов), **Дохода**.

### Тип статьи

- **Денежная** («на дату баланса») — Обязательства, некоторые Активы (счета денежных средств, дебиторская и кредиторская задолженность, займы, финансовые вложения).
- **Неденежная** («на дату операции») — Доходы, Расходы, Капитал, часть Активов (запасы, ОС, НМА).

### Иерархия

Категория → Общая статья (I) → Общая статья (II) → Статья

## Системные и пользовательские статьи

**Системная статья** — обязательная статья для работы отчётов, баланса и базовой логики ПланФакта. Помечается 🔒 и/или 🚫. Нельзя удалять; в ряде случаев нельзя редактировать, менять категорию, родительскую статью или структуру подстатей.

**Пользовательская статья** — статья, которой управляет пользователь. Можно редактировать и удалять или перемещать в иерархии.

**Правило для агента:** если статья помечена 🔒 и/или 🚫 или в её свойствах есть ограничения — трактовать как системную и не предлагать её удаление, перенос в другую категорию или изменение базовой роли в отчётности.

## Обозначения

- 🔒 — системная статья: нельзя редактировать / удалять / менять категорию
- 🚫 — скрыта из UI пользователя (`BaseViewRestriction`); присутствует в API-ответах
- **Денежная** / **Неденежная** — тип статьи, влияет на доступные даты в операции

## Флаги `specialAttributes`

В API-ответе каждая статья может содержать поле `specialAttributes` — массив строк-ограничений. Агент использует эти флаги, чтобы понять, к какому типу операции можно назначить статью.

| Флаг                                                         | Что означает                                                            | Применять?                                                            |
| ------------------------------------------------------------ | ----------------------------------------------------------------------- | --------------------------------------------------------------------- |
| `BaseViewRestriction`                                        | Статья скрыта из UI; в API присутствует                                 | ✅                                                                     |
| `SelectOperationRestriction`                                 | Нельзя назначить ни одному типу операции (контейнер или категория)      | ✅                                                                     |
| `SelectParentOperationCategoriesRestriction`                 | Нельзя назначить операции; родительская/групповая статья                | ✅                                                                     |
| `SelectIncomeOperationRestriction`                           | Нельзя назначить Поступлению                                            | ✅                                                                     |
| `SelectOutcomeOperationRestriction`                          | Нельзя назначить Выплате                                                | ✅                                                                     |
| `SelectAccrualOperationRestriction`                          | Нельзя назначить ни одному Начислению                                   | ✅                                                                     |
| `SelectAccrualOutcomeByIncomeOperationRestrictionForOutcome` | Нельзя назначить расходной части Начисления                             | ⚠️ зависит от контекста операции — не применять как общее ограничение |
| `SelectAccrualOutcomeByIncomeOperationRestrictionForCapital` | Нельзя назначить капитальной части Начисления                           | ⚠️ зависит от контекста операции — не применять как общее ограничение |
| `SelectAccrualIncomeOperationRestriction`                    | Нельзя назначить доходной части Начисления                              | ✅                                                                     |
| `SelectAccrualOutcomeOperationRestriction`                   | Нельзя назначить расходному Начислению                                  | ⚠️ зависит от контекста операции — не применять как общее ограничение |
| `FilterOperationRestriction`                                 | Нельзя использовать в фильтрах операций                                 | ✅                                                                     |
| `UsageOperationRestriction`                                  | Нельзя назначить операции через стандартный UI                          | ✅                                                                     |
| `UsageOperationCategoryRestriction`                          | Нельзя выбрать в справочнике статей операции                            | ✅                                                                     |
| `SelectCalculationDateRestriction`                           | Доступна только дата оплаты; дата начисления для этой статьи недоступна | ✅                                                                     |

**Правила для агента:**

- Если у статьи есть `SelectOperationRestriction` или `SelectParentOperationCategoriesRestriction` — нельзя назначить операции.
- Если есть `SelectIncomeOperationRestriction` — нельзя использовать у Поступления.
- Если есть `SelectOutcomeOperationRestriction` — нельзя использовать у Выплаты.
- Если есть `SelectCalculationDateRestriction` — при создании операции не передавать дату начисления, даже если тип статьи формально Неденежная.
- Статьи с `UsageOperationRestriction` + `UsageOperationCategoryRestriction` + `BaseViewRestriction` — полностью технические; не использовать в операциях и не показывать пользователю.

**Флаги, которые нужно игнорировать:**

Следующие флаги присутствуют в API-ответе, но агент их **не применяет**:

- `SelectAccrualOutcomeByIncomeOperationRestrictionForOutcome`
- `SelectAccrualOutcomeByIncomeOperationRestrictionForCapital`
- `SelectAccrualOutcomeOperationRestriction`

Если эти флаги есть у статьи — они не ограничивают её использование в Начислениях. Считать статью доступной для Начисления по общим правилам.

## Ключевые правила для агента

### Для P&L (ОПУ / отчёт о прибылях и убытках)

- Доходная часть: `operationCategoryType == Income` (статьи группы 1)
- Расходная часть: `operationCategoryType == Outcome` (статьи группы 2)
- Активы, Обязательства, Капитал в P&L не участвуют

### Для выручки

- По умолчанию: 1.1 + 1.2 (Нераспределённый доход + Выручка от продаж)
- 1.3 Прочие доходы **не включать** без явного запроса
- Assets, Capital, Liabilities — никогда не включать в выручку

### Для балансовых расчётов

- Денежные статьи учитываются на дату баланса (as-of date)
- Неденежные статьи учитываются на дату операции
- Статьи 3.1.2 (Дебиторка) и 4.1.1 (Кредиторка) — основа для задолженности

### Для operationCategoryType в API

- Операции могут принадлежать к Income, Outcome, Assets, Liabilities, Capital
- При accrual-аналитике смотреть `operationParts[].operationCategoryType`, а не только верхний `operationType`
- Части одной операции могут иметь разные типы (например, Assets + Liabilities при проводках)

### Системные и скрытые статьи

- Статьи с 🚫 (`BaseViewRestriction`) не отображаются пользователю в UI, но присутствуют в API-ответах
- Статьи 3.1.1.1–3.1.1.5 (типы счетов денежных средств) системные — их не показывать в пользовательских отчётах как статью операции
- 3.1.4.3 «Отправленные платежи» и 4.1.2.3 «Полученные платежи» — технические парные статьи для Move-операций

### Нельзя выбрать в операции

Полный список флагов-ограничений и их значения — в секции «Флаги `specialAttributes`» выше.

Ключевые случаи:

- `SelectOperationRestriction` / `SelectParentOperationCategoriesRestriction` — нельзя назначить ни одному типу операции
- `SelectIncomeOperationRestriction` — нельзя назначить Поступлению
- `SelectOutcomeOperationRestriction` — нельзя назначить Выплате
- `SelectAccrualOperationRestriction` — нельзя назначить ни одному Начислению
- `SelectCalculationDateRestriction` — не передавать дату начисления при создании операции с этой статьёй

## Быстрая группировка статей

`GET /api/v1/operationcategories/group-list` возвращает готовую иерархию статей по группам `Income`, `Outcome`, `Assets`, `Liabilities`, `Capital`.

Использовать, когда агенту нужно быстро определить группу статьи или найти системные нераспределённые статьи без ручной сборки дерева из полного `/operationcategories`.

Это справочный endpoint. Он не возвращает операции, суммы, выручку, расходы или остатки.
