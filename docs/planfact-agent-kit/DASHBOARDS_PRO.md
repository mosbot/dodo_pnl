# ПланФакт — Раздел «Показатели ПРО» для AI-агента

Этот файл описывает бизнес-логику раздела **«Показатели ПРО»** (платный модуль) и 14 endpoint'ов `/api/v2/dashboardpro/*`.

`README.md` — policy, router.
`DASHBOARDS.md` — базовый раздел «Показатели», общедоступный.
`REPORTS_V2.md` — ОПУ и ДДС v2.
`BALANCE.md` — табличный Баланс.
`BUDGETS.md` — бюджеты БДДС и БДР (сущности бюджета).
`USE_CASES.md` — бизнес-семантика, method + scope.

## Общее

- Доступен всем пользователям, **но только если в аккаунте действует модуль «Показатели ПРО»**.
  - Для расчётов используются методы из группы `/api/v2/dashboardpro/*` (далее — `DashboardProV2`).
- Если модуль **не доступен** — при попытке пользователя запросить данные из этого раздела агент должен сообщить, что не может вывести данные, так как у пользователя не куплена подписка на модуль «Показатели ПРО».
  - Если пользователь пытается обратиться к ним напрямую — агент должен трактовать это как недоступность штатного PRO-сценария и объяснить причину пользователю.
  - Если пользователь сам описал формулу расчёта — возможность расчёта по пользовательской формуле требует согласования (**НУЖНО СОГЛАСОВАНИЕ**); такой результат не считать штатным ответом PRO-модуля.
- Если доступ к модулю заранее неизвестен, не делать отдельный preflight-запрос только для проверки тарифа. Использовать уже известный account/tariff-контекст, если он есть; иначе первый необходимый `DashboardProV2` вызов одновременно является проверкой доступа. Если он возвращает ошибку доступа к модулю — зафиксировать, что штатный PRO-сценарий недоступен, и сообщить пользователю об ограничении.
- Раздел состоит из 7 подразделов-вкладок: **Движение денег**, **Прибыль**, **Проекты**, **Сделки**, **Баланс**, **Долги**, **Бюджеты**.
- На многих блоках есть переключатель **«Динамика»** — показывает изменение значений относительно предыдущего периода (варианты: Без динамики, Абсолютная, В процентах).
- Если данных недостаточно — некоторые графики отображают заглушку: «Наполните ПланФакт платёжными операциями, и здесь появится реальный график» с демо-данными на фоне.

**Request scope:** слово `дашборд`, `Показатели ПРО`, `локальная страница` или `визуализация` не означает автоматически все 14 endpoint'ов раздела.

- Вызывать только endpoint'ы блоков, которые пользователь реально запросил.
- Для полного экрана вызывать по одному запросу на каждый distinct endpoint из нужных блоков.
- Блоки, которые используют один endpoint, рендерить из одного ответа.
- Delivery (`HTML`, график, таблица, browser-ready файл) не добавляет API-вызовов.
- Если пользователь просит дашборд / виджет / блок из **«Показателей ПРО»** и указывает фильтры раздела (период, ЮЛ/счета, проекты, сделки, поток, метод учёта), сначала вызывать профильный `/api/v2/dashboardpro/*` endpoint. Не переходить к `operations/list`, `reports/*`, `bizinfos/*` или ручной реконструкции до проверки профильного DashboardProV2 endpoint и payload.
- В payload `DashboardProV2` использовать плоскую форму без обёртки `filter`. Имена полей и обязательные поля зависят от конкретного endpoint: например, `accountbalance` требует `currentDate`, `periodStartDate`, `periodEndDate` и использует `accountIds`; `cashflow` использует `accountId`, `projectId`, `method`.

Профильный `/api/v2/dashboardpro/*` endpoint — это primary source для блока PRO. Не добавлять `reports/*`, `bizinfos/*` или `/operations` для контрольной сверки, если профильный endpoint уже покрывает сценарий.

## 1. Движение денег

**Фильтры:** Период, Юрлица и счета, Проекты, Сделки.

**Настройки отображения (вверху справа):**

- Отображение: По месяцам / По кварталам / По годам.
- Поток: Общий / Операционный / Инвестиционный / Финансовый.

### 1.1 Остатки на счетах, <Валюта аккаунта>

- Линейный график с динамикой остатка по всем счетам.
- Ось X — конкретные даты (не зависит от настройки «Отображение»).
- В легенде: «Общий остаток» и каждый счёт с указанием юрлица.
- Если по счёту не было ни одной операции или остаток за период всегда был = 0 — он не выводится.
- **Endpoint:** `POST /api/v2/dashboardpro/accountbalance`.

### 1.2 Фактические остатки на счетах на конец периода, <Валюта аккаунта>

- Таблица:
  - колонки — периоды (месяцы / кварталы / годы);
  - строки:
    - «Остатки на конец периода» с суммой по каждому периоду;
    - Юрлица со своими счетами с суммой по каждому периоду.
- **Endpoint:** `POST /api/v2/dashboardpro/accountbalance`.

### 1.3 Денежный поток, <Валюта аккаунта>

- Показатели: Поступления, Выплаты, Разница — каждый выводится как факт + план.
- Настройки блока:
  - Динамика (Без динамики / Абсолютная / В процентах);
  - План: Отображать план / Скрыть план.
- График: столбцы Поступления (синие) и Выплаты (жёлтые), линия Разница.
  - Если Разница < 0 в каком-то периоде — точка на линии выделяется красным.
- **Endpoint:** `POST /api/v2/dashboardpro/cashflow`.

### 1.4 Горизонтальный анализ поступлений, <Валюта аккаунта>

- Легенда: суммы и % от общего по каждой группе статей поступлений (Доходы, Кредиты, Другие оборотные, Другие краткосрочные, Другие статьи капитала и т.д.).
- График: столбцы по периодам, каждый столбец разбит по группам статей (разные оттенки синего / голубого).
- **Endpoint:** `POST /api/v2/dashboardpro/paymentstructure-cash-flow`.

### 1.5 Горизонтальный анализ выплат, <Валюта аккаунта>

- Легенда: суммы и % по каждой группе статей выплат (Расходы, Запасы, Кредиты, Другие оборотные, Другие краткосрочные, Дивиденды, Другие статьи капитала и т.д.).
- График: столбцы по периодам, каждый столбец разбит по группам статей (разные оттенки оранжевого).
- **Endpoint:** `POST /api/v2/dashboardpro/paymentstructure-cash-flow`.

### 1.6 Горизонтальный анализ поступлений и выплат, <Валюта аккаунта>

- Таблица: колонки — периоды; строки — Поступления, Выплаты, Общий денежный поток.
  - Поступления и Выплаты можно раскрыть до отдельных статей (кнопка «+»).
  - Для каждого периода отображается динамика: абсолютное изменение со стрелкой вверх / вниз.
- Настройки таблицы:
  - Динамика: Абсолютная / В процентах;
  - Чекбокс «Выделять ячейки с откл. больше» — выделяет цветом ячейки с отклонением больше указанного значения.
- **Endpoint:** `POST /api/v2/dashboardpro/paymentstructure-cash-flow`.

## 2. Прибыль

**Фильтры:** Период, Метод учёта (Кассовый / Начисление), Юрлица и счета, Проекты, Сделки.

**Настройки отображения:** По месяцам / По кварталам / По годам.

### 2.1 Чистая прибыль, <Валюта аккаунта>

- Общая сумма чистой прибыли: факт + план.
- Настройка: Динамика.
- График: столбцы по периодам.
  - Положительная прибыль — зелёные столбцы;
  - Отрицательная прибыль — красные столбцы.
- **Endpoint:** `POST /api/v2/dashboardpro/businessprofit`.

### 2.2 Доходы и расходы, <Валюта аккаунта>

- Показатели: Доходы (факт + план), Расходы (факт + план), Чистая прибыль (факт + план).
- Настройки блока: Динамика, План (Отображать / Скрыть).
- График: столбцы Доходы (синие) и Расходы (жёлтые), линия Чистая прибыль.
- **Endpoint:** `POST /api/v2/dashboardpro/businessprofit`.

### 2.3 Рентабельность чистой прибыли, %

- Линейный график по периодам.
- Линии: Рентабельность по чистой прибыли + Средняя рентабельность по чистой прибыли (пунктир).
- Отрицательные значения выделяются красным.
- Формулы:
  - Рентабельность (за период столбца) = (Доходы − Расходы) / Доходы × 100%;
  - Средняя рентабельность = (Доходы − Расходы) / Доходы × 100% за весь выбранный период.
- **Endpoint:** `POST /api/v2/dashboardpro/businessprofit`.

### 2.4 Рентабельность операционной прибыли, %

- Линейный график по периодам.
- Линии: Рентабельность по операционной прибыли + Средняя (пунктир).
- Отрицательные — красным.
- Формула (за период столбца):
  - Рентабельность = (Выручка − Прямые расходы − Косвенные расходы) / Выручка × 100%.
    - Выручка = сумма всех доходов, кроме Прочих доходов;
    - Прямые расходы = сумма всех расходов с типом «Прямые / Переменные»;
    - Косвенные расходы = сумма всех расходов с типом «Косвенные / Постоянные».
- Формула средней рентабельности: те же компоненты, но за весь выбранный период.
- **Endpoint:** `POST /api/v2/dashboardpro/businessprofit`.

### 2.5 Рентабельность валовой / маржинальной прибыли, %

- Линейный график по периодам.
- Линии: Рентабельность валовой / маржинальной прибыли + Средняя (пунктир).
- Отрицательные — красным.
- Формула (за период столбца):
  - Рентабельность = (Выручка − Прямые расходы) / Выручка × 100%.
    - Выручка = сумма всех доходов, кроме Прочих доходов;
    - Прямые расходы = сумма всех расходов с типом «Прямые / Переменные».
- Средняя рентабельность — те же компоненты за весь период.
- **Endpoint:** `POST /api/v2/dashboardpro/businessprofit`.

### 2.6 Горизонтальный анализ доходов, <Валюта аккаунта>

- Легенда: суммы и % по каждой статье доходов.
- График: столбцы по периодам, каждый разбит по статьям доходов (оттенки синего).
- **Endpoint:** `POST /api/v2/dashboardpro/paymentstructure-income-expenses`.

### 2.7 Горизонтальный анализ расходов, <Валюта аккаунта>

- Легенда: суммы и % по каждой статье расходов.
- График: столбцы по периодам, каждый разбит по статьям расходов (оттенки оранжевого).
- **Endpoint:** `POST /api/v2/dashboardpro/paymentstructure-income-expenses`.

### 2.8 Горизонтальный анализ доходов и расходов, <Валюта аккаунта>

- Таблица: колонки — периоды; строки — Доходы, Расходы, Чистая прибыль.
  - Доходы и Расходы раскрываются до статей (кнопка «+»).
  - По каждому периоду — абсолютная динамика со стрелкой.
- Настройки: Динамика, чекбокс «Выделять ячейки с откл. больше».
- **Endpoint:** `POST /api/v2/dashboardpro/paymentstructure-income-expenses`.

### 2.9 Вертикальный анализ доходов и расходов, <Валюта аккаунта>

- Две круговые диаграммы: Доходы и Расходы.
  - В центре каждой — общая сумма.
  - Рядом — легенда со статьями, суммами и % от итога.
- Таблица ниже: колонки — периоды; строки — статьи доходов и расходов в виде % от итога.
  - Показывает долю каждой статьи в общем объёме по каждому периоду.
  - По каждому периоду — динамика изменения доли.
- **Endpoint:** `POST /api/v2/dashboardpro/paymentstructure-income-expenses`.

## 5. Баланс

**Фильтры:** Период, Юрлица.

**Настройки отображения:**

- Отображение: По месяцам / По кварталам / По годам;
- Прибыль: Кассовый метод / Метод начисления.

### 5.1 Собственный капитал, <Валюта аккаунта>

- Столбчатый график по периодам (синие столбцы).
- Настройка: Динамика.
- Формула: **Собственный капитал = Активы − Обязательства**.
- **Endpoint:** `POST /api/v2/dashboardpro/liquidityratios`.

### 5.2 Рентабельность собственного капитала (ROE), %

- Линейный график по периодам.
- Линии: ROE + Средний ROE (пунктир).
- Отрицательные значения — красным.
- Формулы:
  - ROE = Чистая прибыль / |Средний собственный капитал| × 100%.
    - Чистая прибыль = Доходы − Расходы.
      - В прошлом и текущем периоде считаем только по подтверждённым датам, на будущее не строим.
    - Средний собственный капитал = |Средняя стоимость активов − Средняя сумма обязательств|.
      - Средняя стоимость активов = ∑ Активов по дням / (Число дней периода + 1);
      - Средняя сумма обязательств = ∑ Обязательств по дням / (Число дней периода + 1).
  - Среднее ROE = ∑ Чистой прибыли (за весь выбранный период) / |Средний собственный капитал (за весь выбранный период)| × 100%.
    - Все расчёты аналогичны ROE, только за весь выбранный период, а не по дням / месяцам и т.д.
- **Endpoint:** `POST /api/v2/dashboardpro/rentabilityofbalance`.

### 5.3 Выручка и активы, <Валюта аккаунта>

- Показатели: общие суммы Выручки и Активов.
- Настройка: Динамика.
- График: столбцы Выручка (синие) и Активы (оранжевые) по периодам.
- **Endpoint:** `POST /api/v2/dashboardpro/workingcapital`.

### 5.4 Рентабельность активов (ROA), %

- Линейный график.
- Линии: ROA + Средний ROA (пунктир).
- Отрицательные значения — красным.
- Формулы (аналогично ROE, но через активы):
  - ROA = Чистая прибыль / |Средняя стоимость активов| × 100%.
    - Чистая прибыль = Доходы − Расходы; в прошлом и текущем периоде считаем только по подтверждённым датам, на будущее не строим.
    - Средняя стоимость активов = ∑ Активов по дням / (Число дней периода + 1).
  - Среднее ROA = ∑ Чистой прибыли (за весь период) / |Средняя стоимость активов (за весь период)| × 100%.
- **Endpoint:** `POST /api/v2/dashboardpro/rentabilityofbalance`.

### 5.5 Оборотный капитал (общий), <Валюта аккаунта>

- Столбчатый график по периодам (голубые столбцы).
- Настройка: Динамика.
- Формула: **Оборотный капитал = Дебиторка + Запасы − Кредиторка**.
- **Endpoint:** `POST /api/v2/dashboardpro/workingcapital`.

### 5.6 Оборотный капитал (разбивка по составляющим), <Валюта аккаунта>

- Столбчатый график по периодам.
- Каждый столбец разбит по составляющим: Дебиторка (золотая), Кредиторка (красная), Запасы (синяя).
- **Endpoint:** `POST /api/v2/dashboardpro/workingcapital`.

### 5.7 Оборачиваемость, дни

- Линейный график по периодам.
- Линии: Оборачиваемость дебиторки, Оборачиваемость запасов, Оборачиваемость кредиторки.
- Формулы:
  - **Оборачиваемость дебиторской задолженности = Число дней × Средняя дебиторка / Выручка**.
    - Число дней — количество дней в периоде (день → 1, неделя → 7, месяц → кол-во дней в месяце, квартал → кол-во дней в квартале, год → 365).
    - Средняя дебиторка = ∑ Общей ДЗ по дням / (Число дней периода + 1).
    - Выручка — сумма операций Поступлений, Начислений, Отгрузок с подтверждённой датой начисления или оплаты (в зависимости от фильтра «Прибыль») по статьям Дохода, кроме Прочего дохода.
  - **Оборачиваемость запасов = Число дней × Средние запасы / Себестоимость реализованных запасов**.
    - Число дней — то же.
    - Средние запасы = ∑ Запасов по дням / (Число дней периода + 1).
    - Себестоимость реализованных запасов = ∑ сумм операций начислений из статьи «Запасы» в «Расходы».
  - **Оборачиваемость кредиторской задолженности = Число дней × Средняя кредиторка / Основные расходы**.
    - Число дней — то же.
    - Средняя кредиторка = ∑ Общей КЗ по дням / (Число дней периода + 1).
    - Основные расходы — сумма операций Выплат, Начислений, Поставок с подтверждённой датой начисления или оплаты (в зависимости от фильтра «Прибыль») по статьям Постоянных и Переменных расходов (или Прямых и Косвенных).
    - Кредиторка всегда отображается ниже 0, но значение должно быть положительным.
- **Endpoint:** `POST /api/v2/dashboardpro/workingcapital`.

### 5.8 Калькулятор дивидендов

Таблица с расчётом доступных дивидендов. Поля:

- Остаток на счетах — можно указать своё значение.
- (минус) Рост оборотного капитала — можно указать своё значение.
- (минус) Выплата по кредитам — можно указать своё значение.
- (минус) Вложения во внеоборотные активы — можно указать своё значение.
- (минус) Неснижаемый остаток — можно указать своё значение.
- Доступные деньги — всегда вычисляемое поле.
- Чистая прибыль — можно указать своё значение.
- Дивиденды (%, сумма) — можно указать %; сумма всегда вычисляемая.
- Инвестиции в бизнес (%, сумма) — можно указать %; сумма всегда вычисляемая.
- Дивиденды к выплате — всегда вычисляемое поле.
- Оставшиеся дивиденды — всегда вычисляемое поле.
- Доступно для займа — всегда вычисляемое поле.

**Формулы:**

- **Остаток** — общий текущий остаток по счетам на сегодня (по счетам выбранных ЮЛ или по всему бизнесу).
- **Рост оборотного капитала**:
  - Дельта = Оборотный капитал на конец выбранного периода − Оборотный капитал на начало выбранного периода;
  - Оборотный капитал = Дебиторка + Запасы − Кредиторка.
- **Выплаты по кредитам**: сумма по всем плановым операциям Выплат (по дате оплаты от сегодня и в будущее) по статье «Кредиты» на следующий за отчётным период.
  - Если выбран текущий месяц (Март) — следующий период Апрель;
  - Если выбран 1-й квартал (Январь–Март) — следующий период Апрель–Июнь;
  - Если выбран Март + Апрель — следующий период Май + Июнь.
- **Вложения во внеоборотные активы**: сумма по всем плановым операциям Выплат (по дате оплаты от сегодня и в будущее) по всем статьям «Внеоборотных активов» на следующий за отчётным период.
- **Неснижаемый остаток** — по умолчанию 0.
- **Доступные деньги**:
  - Плановые выплаты = Дельта Оборотного капитала + Плановые кредиты + Плановые основные средства + Неснижаемый остаток.
  - Если Плановые выплаты < 0, то **Доступные деньги = Остаток**;
    - если Остаток < 0, то **Доступные деньги = 0**.
  - Если Плановые выплаты ≥ 0:
    - если Остаток ≥ Плановых выплат — **Доступные деньги = Остаток − Плановые выплаты**;
    - если Остаток < Плановых выплат — **Доступные деньги = 0**.
- **Чистая прибыль** = Доходы − Расходы.
- **Дивиденды** = Чистая прибыль × указанный % в Дивидендах.
- **Инвестиции в бизнес** = Чистая прибыль × указанный % в Инвестициях.
- **Дивиденды к выплате**:
  - Если Доступные деньги ≥ Дивидендам → Дивиденды к выплате = Дивидендам;
  - Если Доступные деньги < Дивидендам → Дивиденды к выплате = Доступным деньгам.
- **Оставшиеся дивиденды** = Дивиденды − Дивиденды к выплате.
- **Доступно для займа** = Доступные деньги − Дивиденды к выплате.

**Endpoint:** `POST /api/v2/dashboardpro/dividends`.

### 5.9 График расчёта дивидендов, <Валюта аккаунта>

- Горизонтальный waterfall-chart.
- Визуально отображает составляющие калькулятора дивидендов.
- **Endpoint:** `POST /api/v2/dashboardpro/dividends`.

### 5.10 Коэффициент текущей ликвидности

- Линейный график по периодам.
- Линии: Коэффициент текущей ликвидности + Нижний порог (пунктир) + Верхний порог (пунктир).
- Пороги:
  - верхний — **2,5** (если коэф. выше — точка жёлтая);
  - нижний — **1,5** (если коэф. ниже — точка красная).
- Формула: **Коэф. тек. ликв. = Оборотные активы / Краткосрочные обязательства**.
- **Endpoint:** `POST /api/v2/dashboardpro/liquidityratios`.

### 5.11 Коэффициент быстрой ликвидности

- Линейный график по периодам.
- Линии: Коэффициент быстрой ликвидности + Нижний порог + Верхний порог.
- Пороги:
  - верхний — **1,0** (выше — точка жёлтая);
  - нижний — **0,8** (ниже — точка красная).
- Формула: **Коэф. быстр. ликв. = (Оборотные активы − Запасы) / Краткосрочные обязательства**.
- **Endpoint:** `POST /api/v2/dashboardpro/liquidityratios`.

### 5.12 Коэффициент абсолютной ликвидности

- Линейный график по периодам.
- Линии: Коэффициент абсолютной ликвидности + Нижний порог + Верхний порог.
- Пороги:
  - верхний — **0,5** (выше — точка жёлтая);
  - нижний — **0,2** (ниже — точка красная).
- Формула: **Коэф. абсол. ликв. = Денежные средства на счетах / Краткосрочные обязательства**.
- **Endpoint:** `POST /api/v2/dashboardpro/liquidityratios`.

### 7.1 Бюджет доходов и расходов (БДР), <Валюта аккаунта>

## Правила для агента

- **Проверка модуля перед первым вызовом `DashboardProV2`:**
  - если уже известно, что модуль «Показатели ПРО» недоступен — агент не вызывает ни один `/api/v2/dashboardpro/*`;
  - пользователю честно сообщить, что модуль платный и не подключён;
  - если пользователь описал собственную формулу — расчёт по такой формуле запрещён без явного согласования.
  - если доступ неизвестен, не делать отдельный preflight-запрос только для проверки доступа; первый необходимый `DashboardProV2` вызов считать проверкой доступа и остановиться при ошибке модуля.
- **Routing между `DASHBOARDS.md` и `DASHBOARDS_PRO.md`:**
  - «дашборд», «показатели», «как в ПланФакте» без углублённой аналитики → `DASHBOARDS.md` (`bizinfos/*` + `dashboards/accountbalance` для остатков в базовых «Показателях»);
  - ROE, ROA, ликвидность (текущая / быстрая / абсолютная), оборотный капитал, оборачиваемость, собственный капитал, дивиденды (калькулятор), долги по контрагентам, БДР / БДДС дашборд → **`DASHBOARDS_PRO.md`**.
- **Routing между `DashboardProV2` и отчётами v2:**
  - отдельная аналитическая метрика для dashboard-экрана → `DashboardProV2`;
  - полноценный отчёт ОПУ / ДДС с детализацией по статьям / проектам / сделкам → `POST /api/v2/reports/opu`, `POST /api/v2/reports/dds` (см. `REPORTS_V2.md`);
  - табличный Баланс на одну или две даты → `POST /api/v1/accounting/balance/table` (см. `BALANCE.md`). В `DashboardProV2` нет аналога Баланса в табличной форме.
- **Fallback из DashboardProV2:** неожиданный нулевой ответ, даты `0001-01-01` или пустая детализация — сначала диагностировать payload и фильтры профильного endpoint. Ручной расчёт из `operations/list` допустим только после ошибки доступа/валидации, явного неподдерживаемого фильтра или подтверждения пользователя.
- **Метод учёта (Кассовый / Начисление):** обязателен для тех блоков DashboardProV2, где payload поддерживает переключатель метода учёта: «Прибыль», «Проекты», «Сделки» и отдельные PRO-блоки вкладки «Баланс». Это не меняет общее правило для табличного Баланса из `BALANCE.md`: там `cash / accrual` не спрашивать. Если метод для PRO-блока не указан пользователем — 1 уточняющий вопрос до первого API-запроса (см. `README.md`).
- **Единый dataset для вкладки:** внутри одной вкладки блоки часто используют один и тот же endpoint (`businessprofit`, `paymentstructure-*`, `workingcapital`, `liquidityratios`). Делать один POST на endpoint и рендерить все блоки вкладки из одного ответа — не вызывать endpoint повторно под каждый блок.
- **Динамика:** переключатель «Без динамики / Абсолютная / В процентах» — это client-side трансформация уже полученного dataset; не считать её поводом для дополнительного API-вызова.
- **Формат дат:** `YYYY-MM-DD` для всех параметров (см. `QUICKSTART.md`).
- **Write-ограничений нет:** весь `DashboardProV2` — read-only (`POST` по HTTP, но без изменения данных). Policy из `README.md` о подтверждении write-запросов к этой группе не применяется.
