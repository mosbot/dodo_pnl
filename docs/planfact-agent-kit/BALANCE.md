# ПланФакт API — BALANCE для AI-агента

Этот файл покрывает Баланс (табличная форма): `POST /api/v1/accounting/balance/table`.

`README.md` — policy, router.
`QUICKSTART.md` — короткий technical index.
`USE_CASES.md` — бизнес-семантика и `method + scope`.
`OPERATION_CATEGORIES.md` — иерархия статей (Assets / Liabilities / Capital).
`REPORTS_V2.md` — ОПУ и ДДС.

## Когда открывать

- пользователь просит Баланс, финансовое положение, состояние бизнеса на дату;
- нужен срез Активы / Обязательства / Капитал на конкретную дату или набор дат;
- пользователь просит «как изменился баланс» за период;
- нужно проверить сходимость баланса (Активы = Обязательства + Капитал).

## Ограничения доступа и неполный результат

- Если `POST /api/v1/accounting/balance/table` возвращает `403` или access-error, считать, что текущий штатный сценарий Баланса недоступен по тарифу, правам или scope.
- Если API возвращает только доступную часть данных по выбранным ЮЛ или счетам, агент должен явно предупредить, что результат может быть неполным.
- Альтернативный расчет Баланса из других источников не считать штатным Балансом API.

## Ключевые концепции

### Баланс — это состояние на дату, не движение за период

- Каждая колонка = срез на конец даты этой колонки («конец дня», `23:59:59` — все операции вплоть до конца дня включены).
- Значения рассчитываются **накопительно с начала ведения учёта** до даты колонки.
- Период в запросе влияет только на **набор колонок**; он не ограничивает расчёт только этим интервалом.

**Примеры:**
- Баланс на одну дату: `periodStartDate = periodEndDate = 2026-01-31` → одна колонка
- Баланс по месяцам за год: `periodStartDate = 2025-01-01`, `periodEndDate = 2025-12-31`, `standardPeriod = Month` → 12 колонок (31.01, 28.02, ...)
- Значение в колонке 31.01.2025 = накопленный итог **с начала учёта** по 31.01.2025, а не «сумма за январь»
- Если нужна динамика «как изменился показатель за месяц» — считать разницу между двумя соседними колонками локально

### Главное тождество

Для каждой колонки: **Активы = Обязательства + Капитал**

Поле `isBalanced` в `BalanceValueByPeriod` показывает, выполняется ли равенство `Активы − Пассивы = 0`. Если `isBalanced == false` — данные расходятся; сообщить пользователю, не выравнивать числа вручную.

### Источник данных

Баланс строится по **проводкам**, не напрямую по операциям. Операции порождают проводки, которые уже влияют на Баланс.

### Нераспределённая прибыль

Для строки `Капитал -> Нераспределённая прибыль` ориентироваться на такую формулу:

`Нераспределенная прибыль = Доходы - Расходы - Дивиденды + Курсовая разница`

Практически:

- `Текущий год` — текущий финансовый результат периода с учётом дивидендов и `courseDiff`;
- `Прошлый период` — накопленный результат прошлых периодов;
- `courseDiff` использовать как компонент строки `Нераспределённая прибыль -> Текущий год`, а не как отдельную самостоятельную статью капитала.

## Endpoint

`POST /api/v1/accounting/balance/table`

Считать **read-only** endpoint'ом: не меняет данные.

### Параметры запроса

| Параметр          | Тип                | Описание                                                                     |
| ----------------- | ------------------ | ---------------------------------------------------------------------------- |
| `agentIds`        | array[int]         | Фильтр по ЮЛ (nullable; если не задан — все ЮЛ)                              |
| `currencyCode`    | string             | Валюта отчёта                                                                |
| `standardPeriod`  | string             | Гранулярность колонок: `Day \| Week \| Month \| Quarter \| HalfYear \| Year` |
| `periodStartDate` | string (date-time) | Начало периода                                                               |
| `periodEndDate`   | string (date-time) | Конец периода                                                                |

Даты: передавать в формате **`YYYY-MM-DD`** — как и для всех остальных endpoints этого стенда. Swagger указывает `date-time`, но на рабочем стенде формат с временем вызывает ошибку. Подробнее → `QUICKSTART.md`, раздел «Формат дат».

### Структура ответа

```json
{
  "data": {
    "periods": ["2025-01-31T23:59:59", "2025-02-28T23:59:59"],
    "assets":      { /* дерево Активов */ },
    "liabilities": { /* дерево Обязательств + Капитала */ }
  }
}
```

- `periods` — массив дат колонок (конец каждого периода)
- `assets` — дерево строк Активов
- `liabilities` — дерево строк Обязательств **и Капитала** (= Пассивы в бухгалтерском смысле)

### TableBalanceItemResponseModel — одна строка дерева

| Поле                  | Тип            | Описание                                 |
| --------------------- | -------------- | ---------------------------------------- |
| `title`               | string         | Название статьи или счёта                |
| `accountCategoryType` | string (enum)  | Тип категории (см. раздел ниже)          |
| `operationCategoryId` | int (nullable) | ID учётной статьи (если строка = статья) |
| `accountId`           | int (nullable) | ID счёта (если строка = счёт)            |
| `valuesByPeriod`      | array          | Суммы по каждой колонке                  |
| `childItems`          | array          | Дочерние строки (рекурсивное дерево)     |

### BalanceValueByPeriod — значение в одной колонке

| Поле             | Тип               | Описание                                                                      |
| ---------------- | ----------------- | ----------------------------------------------------------------------------- |
| `value`          | double (nullable) | Сумма на дату колонки                                                         |
| `courseDiff`     | double (nullable) | Курсовая разница (только для строки «Нераспределённая прибыль → Текущий год») |
| `isBalanced`     | boolean           | `true` = Активы − Пассивы = 0 для этой колонки                                |
| `startDate`      | date-time         | Начало периода колонки                                                        |
| `endDate`        | date-time         | Конец периода колонки                                                         |
| `standardPeriod` | string            | Гранулярность колонки                                                         |

## Структура баланса

`*` ниже означает системную строку / агрегирующий узел дерева.

### Активы (`assets`)

```text
Активы*
├── Оборотные активы*
│   ├── Дебиторская задолженность*
│   │   ├── Денежная*
│   │   └── Неденежная*
│   ├── Денежные средства*
│   │   ├── Наличные счета*
│   │   │   └── Денежные счета с типом Наличный
│   │   ├── Безналичные счета*
│   │   │   └── Денежные счета с типом Безналичный
│   │   ├── Карты физлиц*
│   │   │   └── Денежные счета с типом Карта физлица
│   │   └── Электронные счета*
│   │       └── Денежные счета с типом Электронный
│   ├── Запасы*
│   │   └── Подстатьи
│   ├── Другие оборотные*
│   │   └── Подстатьи (в том числе «Отправленные платежи (перемещения)»)
│   └── Переводы в пути
└── Внеоборотные активы*
    ├── Основные средства*
    │   └── Подстатьи
    └── Другие внеоборотные*
        └── Подстатьи
```

**Итого Активы** = Оборотные + Внеоборотные.

### Обязательства + Капитал (`liabilities`)

```text
Пассивы*
├── Краткосрочные обязательства*
│   ├── Кредиторская задолженность*
│   │   ├── Денежная*
│   │   └── Неденежная*
│   └── Другие краткосрочные*
│       └── Подстатьи
├── Долгосрочные обязательства*
│   ├── Кредиты*
│   │   └── Подстатьи
│   └── Другие долгосрочные*
│       └── Подстатьи
└── Капитал*
    ├── Вложения учредителей*
    │   └── Подстатьи
    ├── Нераспределённая прибыль*
    │   ├── Текущий год*
    │   └── Прошлый период*
    └── Другие статьи капитала*
        └── Подстатьи
```

**Итого Обязательства + Капитал** = Обязательства + Капитал.

> Важно: в API-ответе Капитал (`Capital`) входит в блок `liabilities`, а не является отдельным top-level ключом. Это стандартная бухгалтерская логика: Пассивы = Обязательства + Капитал.

## Правила отображения счетов

### Денежные счета

- Нет операций и нет начального остатка → счёт **не выводится**.
- Есть операции, но итоговая сумма = 0 → счёт **выводится**.

### Архивные счета

- Остатки по архивным счетам — отдельная строка **«Счета в архиве»**, всегда последняя внутри группы.
- Правило действует отдельно для каждого типа: безналичные / наличные / карты физлиц / электронные.

## Как интерпретировать запросы пользователя

| Запрос пользователя    | Что делать                                                   |
| ---------------------- | ------------------------------------------------------------ |
| «Баланс на дату»       | Один срез: `periodStartDate` = `periodEndDate` = нужная дата |
| «Баланс за период»     | Набор колонок: задать период и `standardPeriod`              |
| «Как изменился баланс» | Сравнивать `value` между колонками; не подменять ДДС или ОПУ |
| «Сходится ли баланс»   | Проверить `isBalanced` по нужным колонкам                    |

## Правила для агента

- Баланс ≠ ДДС, ≠ ОПУ, ≠ оборот за период, ≠ движение денег за интервал.
- Баланс — это **состояние (snapshot) на дату**.
- `periods` в ответе — даты колонок, не границы расчётного периода.
- Значения накопительные с начала учёта; не интерпретировать как «сумма за месяц».
- Для навигации по дереву использовать `accountCategoryType`, а не парсить `title`.
- `courseDiff` — курсовая разница; присутствует в строке «Нераспределённая прибыль → Текущий год».
- Для строки `Нераспределённая прибыль` помнить формулу: `Доходы - Расходы - Дивиденды + Курсовая разница`.
- Если `isBalanced == false` — честно сообщить пользователю; не выравнивать числа вручную.
- Не использовать этот endpoint, если пользователю нужен cashflow или ОПУ.

---

## Known caveats

- Единственный endpoint: `POST /api/v1/accounting/balance/table`; нет GET-версии.
- Даты в параметрах запроса — `YYYY-MM-DD`, несмотря на то что swagger указывает `date-time`.
- Капитал (`Capital`) в API-ответе входит в блок `liabilities`, а не является отдельным ключом.
